But I'm already debugging!

"Debugging" is a generic term, but there is also a specific tool called a "debugger".

Allows you to step through code as it is being executed, and examine variables and branching logic as you go.

When to use a debugger

If you find yourself...

  • Dropping several dpm() statements into your code
  • Reloading and reloading and reloading

The debugging stack

  • MAMP, or something to run your web stack
  • XDebug, a PHP debugger
  • NetBeans, an IDE

IDE = Integrated Development Environment

Configuring PHP

Enable it in your php.ini

Configuring PHP in MAMP

MAMP comes with XDebug installed, but you will need to enable it in your php.ini in order to use it.
